Microsoft Agent 365 governs Microsoft agents. Google governs Google agents. ADTP governs all agents — across platforms, across organizations, with cryptographic proof. Open protocol. No vendor lock-in.

The #1 blocker is identity governance. Agents share human credentials. Nobody traces delegation chains. When something goes wrong, there is no cryptographic record of who authorized what.

What ADTP does

Cryptographic trust for every agent, every hop.

01

Verifiable Agent Identity

Every agent gets a cryptographic identity (did:key). No shared credentials. No service accounts.

02

Delegation Chains

When Agent A delegates to Agent B which calls Agent C, every hop is cryptographically signed and traceable to a human sponsor.

03

Provably-Complete Revocation

Compromise one credential, cascade to every descendant. A registration invariant guarantees completeness.

Patent pending
04

Cross-Org Trust

Organization A's agents can securely interact with Organization B's agents. Non-transitive, depth-bounded, bilateral agreements.

Get started

From clone to a registered agent.

bash 
# Clone
git clone https://github.com/Zahanturel/adtp.git

# Build
cd adtp && make build

# Run
./adtpd --config config.yaml
# → "adtpd listening on 127.0.0.1:8080"

# Register an agent
curl -H "Authorization: Bearer <your-api-key>" \
  -X POST localhost:8080/v1/agents \
  -d '{"sponsor_did":"did:key:z6Mk..."}'
