Agent platforms govern their own agents. Zerith governs yours — across platforms, across organizations, with cryptographic proof.

the problem

The identity gap

AI agents inherit human credentials. When Agent A delegates to Agent B, no existing system records that delegation cryptographically or enforces what B can do with that authority.

When something goes wrong — and at scale, it will — there is no verifiable record of who authorized what, to which agent, and what it was allowed to do.

Enterprise agent governance today means vendor-scoped controls. Microsoft for Microsoft. Google for Google. The cross-platform, cross-organization case — where it matters most — has no solution.

the protocol

What ADTP does

Zerith implements ADTP — the Agent Delegation and Trust Protocol. A Go daemon, single binary, that provides four capabilities.

Verifiable agent identity

Every agent gets a cryptographic identity derived from an Ed25519 keypair. No shared credentials. No service accounts. Each agent signs its own operations with its own key.

did:key

Delegation chains

When Agent A delegates to Agent B, ADTP issues a credential encoding who delegated, to whom, what capabilities, under what constraints. RESTRICT mode enforces structural escalation prevention — no descendant can exceed its ancestor's permissions.

UCAN + RESTRICT

Cascade revocation

Revoke one credential. A registration invariant guarantees every descendant is also revoked. Not best effort — provably complete.

Cross-organizational trust

Bilateral agreements between organizations. Non-transitive, depth-bounded. Organization A's agents interact with Organization B's agents under verifiable, auditable policy — without either side exposing its internal IAM.

technical

Built like infrastructure

Go daemon. Single static binary. No runtime dependencies (memory backend). Optional: PostgreSQL. Ed25519 signatures with a planned post-quantum migration path (ML-DSA-65). Thirteen-step credential chain verification. Apache 2.0 open source.

get started

From clone to a registered agent

# Clone and build
git clone https://github.com/Zahanturel/adtp.git
cd adtp && make build

# Run the daemon
./adtpd --config config.yaml
# → adtpd listening on 127.0.0.1:8080
# → generated API key: abc123... ← copy this

# Register an agent (use the API key printed above)
curl -H "Authorization: Bearer <api-key>" \
  -X POST localhost:8080/v1/agents \
  -d '{"sponsor_did":"did:key:z6Mk..."}'
